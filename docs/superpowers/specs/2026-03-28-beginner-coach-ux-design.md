# 杀戮尖塔新手入门教练化改造设计

## 目标

把现有《杀戮尖塔》AI 助手从“会给结论的顾问”改成“会带新手学会选牌和构筑的入门教练”。

本次改造不新增新手/老手模式切换，而是在现有统一界面上直接做教学化、白话化和减法设计。

## 设计原则

### 1. 第一眼先帮用户做决定

首页主区优先解决“我现在该点什么”。

用户打开页面后，应该先看到：

- 现在就点什么
- 为什么
- 不这样选会亏什么

其余内容都服务于这三个问题，不和它们抢注意力。

### 2. 不只是替用户选，还要顺手教会用户判断

每次分析都要顺手回答这三个学习问题：

- 这一步在学什么判断规则
- 这张牌或这个遗物是在补什么
- 接下来两场该继续找什么

目标不是把用户变成“只会按按钮的人”，而是让用户逐步学会自己判断构筑。

### 3. 术语统一改成新手白话

保留原有构筑信息，但主展示改成更容易理解的说法：

- `契合当前构筑` 改成 `和你现在这套牌合不合`
- `route_stage` 主展示改成 `这局现在先学什么`
- `two_fight_goal` 主展示改成 `接下来两场重点找什么`
- `pivot_triggers` 主展示改成 `什么时候再换路线`
- `avoid_now` 主展示改成 `现在先别拿什么`

内部字段可以保持不变，但页面主文案必须白话。

### 4. 给新手兜底，减少“拿错一张就崩”的焦虑

主推荐旁边固定补一个“兜底提示”：

- 如果你拿不准，就先照这个更稳的选

这个提示不需要新增按钮逻辑，只要在文案层明确告诉用户“保守默认解”即可。

### 5. 聊天区要更像会带着问的教练

聊天区不再只是输入框，而是提供常见学习入口：

- 这层该拿牌还是跳过？
- 我现在这套牌缺什么？
- 接下来两场我该找什么？
- 为什么这次更推荐删牌？
- 这回合先出哪几张？

用户点击模板后直接填入输入框或直接发送，降低提问门槛。

## 页面改造范围

### 一、顾问主面板

当前主面板保留，但结构改成更强的“决策优先”布局。

#### 固定保留

- `推荐操作`
- `推荐目标`
- `构筑方向`

#### 新增或强化

- `这次学到的判断规则`
- `不这样做的代价`
- `拿不准时先怎么选`

#### 展示规则

- `原因` 最多展示 2 到 3 条最关键内容
- `备选` 保留，但降权，不抢主推荐注意力
- `把握度` 可以保留，但文案必须白话，如“很稳 / 可直接照做 / 可做但要看后手”

### 二、构筑面板

构筑面板保留，但从“信息汇总卡”改成“构筑教学卡”。

#### 主展示改成以下顺序

1. 这局现在先学什么
2. 接下来两场重点找什么
3. 现在先别拿什么
4. 什么时候再换路线
5. 这套构筑为什么强
6. 这套构筑现在最怕什么

#### 社区来源展示规则

- 社区来源只在一个统一区域展示
- 不在每个备选路线里重复展开同一批来源
- 来源默认显示简短摘要，不做大段堆叠

### 三、聊天与局内记忆

聊天区要从“通用问答”改成“学习入口”。

#### 新增

- 常见问题快捷模板
- 每条回答优先显示 `结论`
- 如果有理由，显示 2 到 3 条 `原因`

#### 行为要求

- 不允许再出现“先按当前局面走稳，这个问题可以继续追问”这种过空回答作为主结果
- 当用户问题能映射到当前界面时，优先返回当前局面的明确建议
- 当用户问“为什么”时，要尽量回答“因为你当前牌组缺什么 / 这张牌补什么 / 不选的代价是什么”

### 四、战斗建议区

战斗建议要更适合新手执行。

#### 主展示固定为三步短句

- 先做什么
- 再做什么
- 什么时候交药

#### 面板继续显示

- 手牌
- 敌人意图
- 当前能量

但主结论优先用人话告诉用户“这一回合该怎么按顺序打”。

## 后端文案改造范围

### 1. 自动建议回复

把自动建议补成更像教学口吻的中文：

- 结论要明确说出“拿哪张 / 跳过 / 删哪张 / 拿哪个遗物 / 先打哪张”
- 原因要尽量落在“当前缺口、当前节奏、后续构筑方向”
- 若有机会成本，必须能直接显示给前端

### 2. 聊天兜底回复

当模型未稳定返回，或退回规则顾问时，不能只给空泛安慰句。

兜底回复至少要满足：

- 有明确结论
- 有至少 2 条原因
- 尽量给出 1 条后续找牌或构筑提示

## 实现边界

### 本次会做

- 前端主推荐区教学化改造
- 构筑卡白话化改造
- 聊天快捷提问模板
- 聊天回答和兜底回答更具体
- 战斗建议文案更短更可执行

### 本次不做

- 新增新手/老手模式切换
- 新增复杂折叠交互或路由页面
- 新增自动操作游戏
- 改动现有后端 API 协议结构

## 验收标准

### 用户体验验收

- 新手第一次打开页面，能在 3 秒内看懂“这一步该点什么”
- 用户能直接看懂“为什么推荐这张牌/这个遗物”
- 用户能从构筑卡里看懂“接下来两场该找什么”
- 用户问“为什么不删牌”“我现在缺什么”时，不再只收到空泛一句话

### 功能验收

- 前端仍能正常构建
- 后端仍能正常编译
- 现有推荐、聊天、构筑、战斗面板数据链路不被破坏

## 设计自检

- 本次目标聚焦在“教学化和人性化”，没有扩到新的系统能力
- 页面改动集中在 `frontend/src/App.vue`
- 文案与兜底逻辑改动集中在 `sts_ai_assistant/service/assistant_service.py` 与 `sts_ai_assistant/service/rule_based_advisor.py`
- 没有引入新的协议字段依赖，优先复用现有数据
